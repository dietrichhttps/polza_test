# Блиц: Мой AI-стек

## IDE и AI-плагины

**Основная IDE:**  
- VS Code с плагинами: GitHub Copilot, Tabnine, Codeium

**Специализированные AI-инструменты:**
- Cursor для экспериментов с AI-ассистентом в коде

- Replit Agent для демо-проектов

## Модели и задачи

**GPT-4 / GPT-4 Turbo:**
- Архитектурные решения и дизайн-паттерны
- Рефакторинг и оптимизация кода
- Написание документации и технических текстов
- Дебаг сложных проблем

**Claude 3.5 Sonnet:**
- Генерация кода (особенно Python/JavaScript)
- Анализ кодовой базы
- Unit-тесты
- Code review

**Llama 3.1 (через Groq):**
- Быстрые подсказки и автодополнение
- Простые рефакторинги
- Генерация boilerplate кода

**o1-preview:**
- Решение сложных алгоритмических задач
- Оптимизация производительности
- Математические вычисления в коде

## MCP (Model Context Protocol)

Использую несколько MCP серверов:

**Базовые:**
- `filesystem-mcp` - работа с файлами
- `git-mcp` - операции с Git
- `web-search-mcp` - поиск актуальной информации

**Специализированные:**
- `database-mcp` - SQL запросы и работа с БД
- `api-testing-mcp` - тестирование API эндпоинтов
- `docker-mcp` - управление контейнерами

**Самописные:**
- `project-structure-mcp` - анализ архитектуры проектов
- `documentation-mcp` - автоматическая генерация доков

## Cursorrules и системные инструкции

**Да, использую регулярно. Самые полезные правила:**

### Качество кода:
```
- Пиши чистый, читаемый код с комментариями на русском
- Следуй PEP 8 для Python и ESLint для JS
- Используй type hints везде где возможно
- Принцип SOLID - не игнорируй
```

### Безопасность:
```
- Никогда не храни ключи и пароли в коде
- Валидируй все входные данные
- Используй HTTPS для API запросов
- Проверяй права доступа к файлам
```

### Архитектура:
```
- Разделяй бизнес-логику и UI
- Используй dependency injection
- Создавай переиспользуемые компоненты
- Думай о масштабировании с самого начала
```
